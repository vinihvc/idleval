# i18n/

> Parent: [src/AGENTS.md](../AGENTS.md) · Stack: Vite + React 19 + Jotai + Paraglide + Ark UI

## Purpose

Internationalization — Paraglide, JSON sources, locale runtime, and localized content hooks.

## Do

- Edit only `messages/{locale}.json` (`en`, `es`, `pt`)
- Keep key parity across all three locale files
- Run `pnpm i18n:check` before committing strings
- Use `m["ui.settings.title"]()` or typed keys via `@/i18n/messages`
- Rich text with `{#tag}…{/tag}` + `<ParaglideMessage markup={…}>`
- Locale via `I18nProvider` + strategy in `paraglide-init.ts` (Jotai `settingsAtom`); `key={locale}` remounts children because Paraglide uses `reload: false`

## Don't

- Edit `i18n/paraglide/` — generated by `pnpm paraglide:compile`
- Add a locale without updating `locale.ts`, `project.inlang/settings.json`, and all JSON files
- Import `game/` here

## Patterns

- Flat dot-notation keys: `ui.nav.upgrades`, `factory.grain.name`
- `localize.ts`: `translate()`, `hasMessageKey()`, `localizeLore()`
- Hooks: `useLocalizedFactory`, `useLocalizedGod` in `hooks/`
- Tests: locale pinned to `en` in `src/test/paraglide-test-setup.ts`; structural validation via `i18n:check`
- Shared Paraglide Vite options: `paraglide.vite.ts` (used by `vite.config.ts` and `vitest.config.ts`)

## Key files

| File | Role |
|------|------|
| `messages.ts` | Re-export compiled `m` |
| `provider.tsx` | `I18nProvider`, `useLocale()` |
| `locale.ts` | Detection, `SUPPORTED_LOCALES`, labels |
| `localize.ts` | Translation helpers for `content/` |
| `paraglide-init.ts` | Client-side locale strategy |
| `paraglide.vite.ts` | Shared `paraglideVitePlugin` options |
| `messages/{locale}.json` | Source of truth for strings |

## Neighbors

- Reads from: `content/` (hooks), `store/atoms/settings` (persisted locale)
- Consumed by: `components/`, `content/`, `app/providers.tsx`
